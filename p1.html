Company Website | Front-end Coding

Overview:

In teams of 2, we designed a company website, with updates/events about the company, how to contact the company, a multi-product listing, a detailed view of a product, the entire process of purchasing a product and appropriate means for the user to provide feedback on a product.
Link to final website

Context, Skills & Tools:

  • Context:

    A Web Design & Development SIAT course at SFU.

  • Skills:

    Designing a responsive website, using HTML, CSS & JavaScript.

  • Tools:

    Sublime & GitKraken / GitHub

Problem or Objective

To create a responsive shopping website, which allows customers to views update/events, learn about the company, appropriate means for the user to provide feedback, view listings for multiple products at a time, view a detailed view of a product & a full purchasing process for the company. Post of this project, we used Git, which was for version control & coordinating work when not working physically with one another.

Description of Solution

We began by looking at the indicaul project we each did in the classes immediately preceding this project. It was to create a style guide for a website, with the same criteria as this project. We opted to use my partners overall style for the website, but my checkout design. From here we filled in each of our pages based on these, while also taking inspiration from newegg.ca for anything that wasn’t included in our initial style guide. For this project I built the checkout, about & contact page. On top of that, I also created the homepage, excluding thee “Showcase” section & the hamburger menu, that is displayed on smaller screens.

Final Project Reflection

From this project I learned that when working in teams, it is much easier to develop a website when you have a clear idea of what the end goal should be (style guide), then to split up work & build the section of the website separately. I also learned that it would of been better to have a partner that is more design focused because I am more focused on how the website will function & interact, rather than how it is visually designed. For this project we were both less visual design focused & our end product suffered for it.
