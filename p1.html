Company Website (IAT 339) - Front-end Coding

Project Information

Overview:

In teams of 2 we designed a company website with updates/events, about the company, how to contact the company, a multi-product listing, a detailed view of a product, the entire process of purchasing a product, appropriate means for the user to provide feedback on a product.
Link to final website

Context, Skills & Tools:

  • Context:

    A Web Design and Development (IAT 339) SIAT course at SFU.

  • Skills:

    Designing a responsive website, using HTML, CSS & JavaScript.

  • Tools:

    Sublime & GitKraken / GitHub

Problem or Objective

To create a responsive shopping website that allows customers to views update/events, learn about the company, appropriate means for the user to provide feedback, view listings for multiple products at a time, view a detailed view of a product & a full purchasing process for the company. Additionally, post of the project was to use Git for version control & coordinating work when not working physically with each other.

Project Solution & Reflection

Description of Solution

We began by looking at the indicaul project we each did in the classes immediately preceding this project. It was to create a style guide for a website, with the same criteria as this project. We opted to use his overall style for the website, but my checkout design. From here we filled in each of our pages based on these & taking inspiration from newegg.ca for any that wasn’t initially included in our style guide. For this project I built the checkout, about & contact page. As well as the homepage excluding thee “Showcase” section & the hamburger menu, that is displayed on smaller screens.

Final Project Reflection

From this project I learned that when working in teams, it is much easier to develop a website when you first have a clear idea of what the end goal should be (style guide), then to split up work & build the section of the website separately. I also learned that it would of been better to find a partner that is more design focused because I am more focus on how the website will actually function & interaction, rather than how well visually designed it is. For this project we were both less visual design focused & our end product suffered for it.
